# Data Mapping Document

This document describes how common Excel EMR columns are mapped into the normalized MySQL schema.

## Patients Table Mapping

| Excel Column (Possible Names) | Database Field | Notes |
| --- | --- | --- |
| Patient ID, ID | patient_id | Optional in Excel. Auto-generated if missing. |
| First Name, First_Name, FName | first_name | Required. Trimmed and title-cased. |
| Last Name, Last_Name, LName | last_name | Required. Trimmed and title-cased. |
| Gender, Sex | gender | Optional. Standardized to Male/Female/Other. |
| DOB, Date of Birth, Birth Date | date_of_birth | Parsed and converted to YYYY-MM-DD. |
| Phone, Phone Number, Mobile, Contact Number | phone_number | Required. Normalized; used for deduplication. |
| Email, Email Address, Mail | email | Optional. Lowercased; deduplicated if repeated. |
| Address, Home Address, Location | address | Optional. Trimmed. |
| Blood Group, Blood Type | blood_group | Optional. Standardized (A+, O-, etc.). |
| Registration Date, Registered On | registration_date | Optional. Defaults to current date if invalid or missing. |

## Visits Table Mapping

| Excel Column (Possible Names) | Database Field | Notes |
| --- | --- | --- |
| Visit ID | visit_id | Optional. Usually auto-generated by database. |
| Patient ID | patient_id | Required for visit ingestion. Foreign key to patients.patient_id. |
| Doctor, Doctor Name, Physician | doctor_name | Required for visit record. |
| Symptoms, Complaint, Notes | symptoms | Optional text field. |
| Visit Date, Consultation Date | visit_date | Required. Parsed and converted to YYYY-MM-DD. |

## Data Transformation Rules

1. Column names are normalized to lowercase with underscores.
2. Known aliases are mapped to canonical schema fields.
3. Date fields are parsed with day-first support and stored as ISO dates.
4. Empty strings are converted to NULL.
5. Duplicate patients are removed based on phone_number and email.
